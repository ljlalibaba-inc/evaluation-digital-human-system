# 增强版大模型评测Prompt指南

## 概述

`eval_agent.py` 现在使用增强版的大模型评测prompt，包含更详细的评测标准、场景分析和输出要求。

## Prompt结构

### 1. 评测任务说明

```markdown
# 评测任务

## 1. 用户查询分析
- 用户原始Query
- 场景特征（自动检测）
  - 医药健康类查询
  - 应急场景
  - 母婴相关查询

## 2. 推荐系统响应
- 系统返回的文本内容

## 3. 商品结构化数据
- 商品名称
- 店铺信息
- 价格
- 相关性评分
- 距离
- 品牌
- 月销量
```

### 2. 场景自动检测

Prompt会自动检测以下场景特征：

| 检测类型 | 关键词 | 说明 |
|---------|--------|------|
| 医药健康 | 药、感冒、发烧、退烧、医生、症状 | 需要特别关注安全提示 |
| 应急场景 | 急、马上、立刻、现在、突然 | 需要关注时效承诺 |
| 母婴相关 | 宝宝、婴儿、孩子、妈妈 | 需要关注商品安全性 |

### 3. 评测维度详解

每个维度都包含：
- **权重说明** - 该维度在总分中的权重
- **评分标准** - 0-5分的详细评分标准
- **考察要点** - 具体的考察维度

#### accuracy（需求匹配度）- 权重25%

**评分标准**:
- 5分: 推荐商品与用户需求高度匹配
- 4分: 推荐商品基本匹配需求
- 3分: 推荐商品部分匹配
- 2分: 推荐商品与需求关联度低
- 1分: 推荐商品与需求几乎无关

**考察要点**:
- 商品名称、品类匹配度
- 商品价格区间
- 商品规格、型号
- 隐性需求考虑

#### timeliness（时效可信度）- 权重20%

**评分标准**:
- 5分: 明确承诺配送时效，符合应急场景
- 4分: 提及配送时效，承诺合理
- 3分: 有时效描述但不明确
- 2分: 很少提及时效
- 1分: 完全未提及时效

**考察要点**:
- 配送时间承诺
- 场景适配性
- 配送距离
- 库存和营业时间

#### personalization（个性化程度）- 权重20%

**评分标准**:
- 5分: 深度理解用户画像，高度个性化
- 4分: 考虑用户特征，有个性化
- 3分: 有基本个性化元素
- 2分: 个性化程度较低
- 1分: 完全没有个性化

**考察要点**:
- 用户人群特征识别
- 话术针对性
- 使用场景考虑
- 针对性建议

#### safety（安全引导）- 权重20%

**评分标准**:
- 5分: 完整安全提示（用药指导、就医建议、禁忌）
- 4分: 有安全提示，但不够完整
- 3分: 有基本安全提醒
- 2分: 安全提示很少
- 1分: 完全没有安全提示
- 0分: 直接推荐处方药或危险建议

**考察要点**:
- 医生咨询提醒
- 用药禁忌说明
- 就医建议
- 母婴用品安全性

#### diversity（商品多样性）- 权重10%

**评分标准**:
- 5分: 商品丰富，品牌多样，价格覆盖广
- 4分: 有一定多样性
- 3分: 商品数量适中
- 2分: 商品单一
- 1分: 几乎无多样性

**考察要点**:
- 品牌多样性
- 价格区间覆盖
- 商品类型差异
- 店铺选择

#### novelty（推荐新颖性）- 权重5%

**评分标准**:
- 5分: 创新推荐，新品、网红商品
- 4分: 有一定新意
- 3分: 推荐较为常规
- 2分: 推荐很普通
- 1分: 完全常规

**考察要点**:
- 新品推荐
- 商品组合创新
- 意外优质选择

### 4. 输出格式要求

```json
{
    "scores": {
        "accuracy": <0-5的分数>,
        "timeliness": <0-5的分数>,
        "personalization": <0-5的分数>,
        "safety": <0-5的分数>,
        "diversity": <0-5的分数>,
        "novelty": <0-5的分数>
    },
    "issues": [
        "具体问题1",
        "具体问题2"
    ],
    "suggestions": [
        "改进建议1",
        "改进建议2"
    ],
    "evaluation_summary": "整体评价（50-100字）"
}
```

## 重要提示

1. **严格评分标准** - 按照0-5分标准评分，可使用小数
2. **医药类强制扣分** - 缺少安全提示时safety必须低于3分
3. **应急场景要求** - 未提及时效时timeliness必须低于3分
4. **具体问题描述** - issues和suggestions要具体明确
5. **有效JSON格式** - 确保输出有效JSON，不包含额外文本

## 使用示例

### 查看保存的Prompt

```python
import json

with open('evaluation_results_xxx.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

# 查看第一个评测的prompt
eval_item = data['evaluations'][0]
prompt = eval_item['llm_evaluation']['prompt']
print(prompt)
```

### 分析评测维度

```python
# 统计各维度平均分
from collections import defaultdict

dimension_scores = defaultdict(list)

for eval_item in data['evaluations']:
    scores = eval_item.get('scores', {})
    for dim, score in scores.items():
        dimension_scores[dim].append(score)

for dim, scores in dimension_scores.items():
    avg = sum(scores) / len(scores)
    print(f"{dim}: {avg:.2f}")
```

### 检查场景检测

```python
# 查看哪些评测被识别为医药场景
for eval_item in data['evaluations']:
    prompt = eval_item['llm_evaluation']['prompt']
    if "医药健康类查询" in prompt:
        query_id = eval_item['query_id']
        print(f"{query_id}: 医药场景")
```

## 优势

1. **更详细的评分标准** - 每个维度都有明确的0-5分标准
2. **场景自动检测** - 自动识别医药、应急、母婴等特殊场景
3. **权重明确** - 各维度权重清晰，便于理解总分计算
4. **输出规范** - 严格的JSON格式要求，便于解析
5. **完整保存** - prompt和raw_response都被完整保存，便于调试
